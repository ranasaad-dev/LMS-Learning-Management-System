# Security Policy

## Supported Versions

We actively monitor and provide security updates for the current major release of the LMS Platform. As this is a modern MERN stack application, we prioritize the latest stable dependencies.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.0   | :white_check_mark: |
| < 1.0   | :x:                |

## Environment & Testing
This project has been fully audited and tested under the following environment to ensure stability and dependency compatibility:
* **Node.js**: v24.14.1
* **npm**: 11.12.0
* **Core Stack**: React 19, Express 5, Mongoose 9, Vite 8

## Reporting a Vulnerability

We value the work of security researchers in keeping our community safe. If you discover a security vulnerability, please follow the steps below to disclose it responsibly.

### How to Report
Please **do not** open a public GitHub issue for security vulnerabilities. Instead, send a comprehensive report to our security team via email.

**Contact Information:**
* **Email:** ranasaad.dev@gmail.com 
*(Constructed by replacing '-' with '.' in the username ranasaad-dev)*

### Disclosure Requirements
To help us triage and fix the issue quickly, please include:
1. **Summary:** A brief description of the vulnerability.
2. **Impact:** How this could affect users or the system.
3. **Reproduction:** Clear instructions or a Proof of Concept (PoC) script.
4. **Environment Details:** Note that the app is tested on Node v24.14.1.

### What to Expect
* **Response Time:** You can expect an initial response within **24 hours**.
* **Accepted Vulnerability:** You will be granted an official appreciation email and **enlisted in our Contributors list**.
* **Rejected Report:** You will receive a thank you email with an apology and a brief explanation.
