# Contributing to LMS Platform

First off, thank you for considering contributing to this LMS project! It’s people like you who make the open-source community such a great place to learn, inspire, and create.

To maintain a high standard of code and documentation, please follow these guidelines.

---

## 🛠 Tech Stack Requirements
Before contributing, ensure your environment matches our production-tested setup:
* **Node.js:** v24.14.1 (or higher)
* **npm:** 11.12.0
* **Frontend:** React 19, Vite 8, Lucide React
* **Backend:** Node.js, Express 5, Mongoose 9 (MongoDB)

---

## 🚀 How Can I Contribute?

### 1. Reporting Bugs
* Check the [Issues Tab](https://github.com/ranasaad-dev/LMS-Learning-Management-System/issues) to see if the bug has already been reported.
* If not, open a new issue. Use a clear title and provide a detailed description, including steps to reproduce and your environment details.
* **Note:** For security vulnerabilities, please refer to our [SECURITY.md](./SECURITY.md).

### 2. Suggesting Enhancements
We are always looking for new features (e.g., Quiz systems, Payment gateways, or UI improvements).
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful for the LMS.

### 3. Pull Requests (PRs)
1.  **Fork** the repository.
2.  **Clone** your fork: `git clone https://github.com/ranasaad-dev/LMS-Learning-Management-System.git`
3.  **Create a Branch:** `git checkout -b feature/your-feature-name` or `bugfix/your-fix-name`
4.  **Install Dependencies:** Run `npm install` in both the root and frontend folders.
5.  **Commit Changes:** Write clear, concise commit messages.
6.  **Push:** `git push origin feature/your-feature-name`
7.  **Open a PR:** Point your PR to the `main` branch of the original repository.

---

## 🎨 Style Guidelines

### Code Style
* Use **Prettier** for code formatting.
* Follow functional component patterns in React (Hooks).
* Ensure all new components are responsive (mobile-friendly).

### Documentation
* If you add a new feature, update the `README.md` if necessary.
* Comment your code where the logic is complex.

### Git Commit Messages
* Use the present tense ("Add feature" not "Added feature").
* Reference issues and pull requests liberally after the first line.
